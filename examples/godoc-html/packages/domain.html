internal/scaffor/domain

Import: github.com/JLugagne/scaffor/internal/scaffor/domain

Overview

Package domain defines the core data structures and contracts for the scaffor template system. It includes template definitions, commands, variables, file operations, and execution options. This package contains the entities and value objects that represent templates and the results of their execution.

Types

Template

▶

Template represents a complete scaffolding template manifest.

type Template struct {
    Name          string            // Template name
    Description   string            // Template description
    Commands      []TemplateCommand // Available commands
    ShellCommands []ShellCommand    // Global shell commands
    Test          []TestStep        // Test steps for validation
    Validate      []string          // Validation commands
}

TemplateCommand

▶

TemplateCommand defines a single command within a template.

type TemplateCommand struct {
    Command      string            // Command name
    Description  string            // Command description
    Variables    []TemplateVariable// Required variables
    Files        []TemplateFile    // Files to create
    PostCommands []string          // Commands to run after
    ShellCommands []ShellCommand   // Shell commands for this command
    Hint         string            // User-facing hint
}

TemplateVariable

▶

TemplateVariable defines a template variable that must be provided at execution time.

type TemplateVariable struct {
    Key         string // Variable name
    Description string // Variable description
}

TemplateFile

▶

TemplateFile specifies a file to be created or modified during template execution.

type TemplateFile struct {
    Source    string // Template source file
    Destination string // Target destination path
    OnConflict string // Conflict resolution strategy
}

ShellCommand

▶

ShellCommand defines a shell command to execute during or after template application.

type ShellCommand struct {
    Command string // Shell command to execute
    Mode    string // Execution mode (all, per-file, etc.)
    Pattern string // Glob pattern for matching files
    Silent  bool   // Suppress output
}

TestStep

▶

TestStep defines a command and parameters to test a template.

type TestStep struct {
    Command string            // Command name to test
    Params  map[string]string // Parameters to provide
}

LintError

▶

LintError represents a validation error found during template linting.

type LintError struct {
    Command string // Command name
    Field   string // Field with error
    Message string // Error message
}

func (e LintError) Error() string // Implements error interface

ExecuteOptions

▶

ExecuteOptions controls the behavior of template execution.

type ExecuteOptions struct {
    DryRun bool // Show what would be done without making changes
    Skip   bool // Skip files that already exist
    Force  bool // Overwrite existing files
}

FileEvent

▶

FileEvent represents a file operation that occurred during template execution.

type FileEvent struct {
    Path   string // File path affected
    Action string // Action performed (created, overwritten, skipped)
}
